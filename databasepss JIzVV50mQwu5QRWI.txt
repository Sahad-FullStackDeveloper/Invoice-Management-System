⚙️ 3️⃣ Explain Features (Core Part)

Break it into modules:

👤 Authentication & Roles
The system has predefined users with role-based access.

Admin has full access,
Sales can manage customers and invoices,
Accountant can handle payments.

I implemented role restriction both in frontend routing and backend authorization.
👥 Customer Management
Users can create, view, and delete customers.
Each customer is linked to invoices.
🧾 Invoice Module
Invoices can be created with customer, amount, and date.

The system automatically calculates:
- Paid amount
- Remaining balance
- Status (Paid / Partial / Unpaid)
💰 Payment Module
Payments are linked to invoices.

When a payment is added or updated, the invoice is automatically updated.

This ensures real-time balance tracking.

🔥 This part is very impressive, don’t skip it.

📊 Dashboard
The dashboard shows total customers, invoices, payments, and revenue.
📄 Pagination
I implemented pagination for invoices and payments to improve performance and user experience.
